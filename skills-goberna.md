# SKILL: GOBERNA SYSTEM (FRONTEND UI/UX)

## FILOSOFÍA DEL PROYECTO

Este producto es:
- una red internacional de consultores políticos
- una plataforma premium
- una interfaz editorial + institucional
- NO es una app genérica
- NO es un SaaS común
- NO es una landing básica

Debe sentirse como:
- LinkedIn + McKinsey + campaña política premium
- elegante, estratégica y limpia
- con jerarquía visual fuerte
- sin ruido visual

## PRINCIPIOS DE DISEÑO

### PROHIBIDO
- colores aleatorios
- sombras exageradas
- UI tipo template barato
- emojis innecesarios
- gradientes chillones
- bordes gruesos
- saturación visual
- iconos inconsistentes

### OBLIGATORIO
- aire (espacio negativo)
- jerarquía clara
- consistencia absoluta
- alineación perfecta
- modularidad

## PALETA DE COLORES

ÚNICOS colores permitidos:
- Amarillo principal: #FFC502
- Azul oscuro: #0F1923
- Negro: #000000
- Negro suave: #0F0F0F
- Blanco: #FFFFFF

REGLAS:
- amarillo = acentos + CTA + verificado
- azul oscuro = texto + bordes
- negro = fondos fuertes
- blanco = fondo principal
- NO usar otros colores.

## TIPOGRAFÍA

ÚNICA fuente: Montserrat

Pesos permitidos:
- Regular (400)
- Medium (500)
- SemiBold (600)
- Bold (700)

JERARQUÍA BASE:
- Hero titles → Bold / ExtraBold (grande)
- Subtítulos → Bold / 24px
- Texto principal → Medium / 16–24px
- Descripción → Medium / 11–14px

## SISTEMA DE BORDES

Regla clave:
- bordes: 0.5px
- color: #0F1923
- radio: suave (consistente)

PROHIBIDO:
- bordes gruesos
- outlines negros duros
- sombras tipo material design agresivas

## SISTEMA DE BOTONES

### TIPOS
**Primario**
- fondo amarillo #FFC502
- texto oscuro #0F1923
- limpio

**Secundario**
- borde blanco o azul
- sin fondo

### REGLAS
- sin efectos exagerados
- hover sutil
- coherencia total

## SISTEMA DE ICONOS
- usar librerías (lucide, react-icons/fa6, etc.)
- estilo consistente
- tamaño uniforme
- PROHIBIDO: mezclar estilos, iconos random, svg deformes

## IMÁGENES

Regla obligatoria:
- TODAS las imágenes deben ser .webp
- PROHIBIDO: jpg final, png final (solo temporal)
- no deformar, mantener proporción, calidad alta

## COMPONENTIZACIÓN

Todo debe ser modular.
PROHIBIDO: componentes gigantes, lógica mezclada

## ANIMACIONES

PERMITIDO:
- micro-interacciones
- hover suaves
- fades
- blur elegante
- shimmer (ej: verificado)

PROHIBIDO:
- animaciones exageradas
- bounce innecesario
- efectos distractores

## ERRORES CRÍTICOS

SI OCURRE → REHACER:
- UI genérica tipo template
- colores fuera de sistema
- no usar Montserrat
- no usar .webp
- romper layout
- no respetar jerarquía
